Tags Framework

Introduction

Tags lets you implement custom tags in Javascript. This simple paradigm makes it possible to quickly bind tags to event handlers, create and manipulate dynamic HTML and implement widgets with a clean HTML interface.

Handling Objects

To begin with, Tags provides a Class mechanism to define object types and instantiate objects. Internally, it is based on Class.js, but Tags provides an interface based on the define() and create() methods:

    • Tags.define({tag:’typeName’, extend:’typeBeingExtended’})

In addition to the ‘tag’ and ‘extend’ attributes, add any additional variables or methods you want to add to the prototype of the Class being defined.

    • var obj = Tags.create({tag:’typeName’});

In addition to the ‘tag’ attribute, add any additional variables or methods you want to add to the object being created. The ‘content’ attribute, like the ‘tag’ attribute has a predefined meaning. It contains any nested objects associated with the object being created. Think of Tag objects as emulating XML in which case the ‘tag’ attribute represents the tag name and the ‘content’ represents nested XML and text. The create() method will recursively invoke the create() method using the object(s) it finds in the ‘content’ attribute.Also, the input to the create() method can be a string, number, boolean or XML Node object. If an XML Node object is passed, it will convert the XML into a Javascript structure. If a string is passed and the string starts with ‘<’, it will be considered XML text and will be parsed and converted into a Javascript structure. If it is a string that does not start with ‘<’ or it is a boolean or number, it will just return the value.

We show lower case names for the ‘tag’ attribute, but tag names are not case sensitive. All tag names will be converted to upper case by both the define() and create() methods.

Handling Views

The Tags Framework also defines a built-in Class called View. The View class has render() and activate() methods. The render() method renders the HTML equivalent of the object and returns it as a jQuery element and the activate method attaches event handlers and performs other tasks which need to be delayed until the tag has been added to the DOM.

A typical way to use a View object is with jQuery such as …

$(“#targetTag”).append(viewObject.render());
viewObject.activate();

What makes the View class useful is that every HTML5 tag name is bound to the View Class. This lets you write something like …

var viewObject = Tags.create("<span>HELLO WORLD</span>");

or the equivalent …

var viewObject = Tags.create({tag:'span',content:'HELLO WORLD'})

As you can see, this gives one a lot of flexibility in creating View components for rendering dynamic HTML.

The next step, of course is to define your own custom tags. Simply define your own render() and activate() methods as needed and you can place your custom tag anywhere a normal HTML tag would go. To clarify, in practice, we override the renderText() method rather than the render() method. Here is a Hello World example …

Tags.define({  
  tag:'hello',
  extend:'view',
  renderText:function() {
    if (this.message) this.content = this.message;
    return this.renderAs('span');
  }
});
...
<body>
  <script type='text/custom-tags'>
    <hello message='Hello World'/>
    <hello>Hello Again</hello>
  </script>
</body>
...

Note that the <script> tag is used to let the Tags framework know that there are custom tags to be handled. It will automatically interpret everything within the <script> tags as XML which is to be passed to the Tags.create() method to create a View object. It then calls the render() method to convert it to a DOM object and inserts it just after the <script> tag. It then invokes the activate() method. So one can simply use custom tags as if they were regular HTML as long as they are contained in an identifying <script> tag with type of “text/custom-tags”.

One nice thing about the Tags Framework is that custom tags are light-weight and easy to define. This means you can create a custom tag just to bind an event handler. Say you had an <input> tag to Start an operation and you wanted to bind it to an event handler. You could do this easily with jQuery, but there is nothing in the <input> tag itself to tell you this is going to happen. And you have to create an ID for the <input>. And where are you going to put that jQuery code? And you have to make sure it gets called. So there are lots of things that can go wrong with a simple operation.

Alternatively, you could create a custom tag called ‘startInput’ which gives you a place to put the event handler. You can look at the HTML and know what is going on. No ID is required and it does not matter where the Javascript code goes. Here is is the ‘startInput’ definition …

Tags.define({
  tag:'startInput',
  extend:'view',
  htmlTag:'input',
  activate: function() {
    this._super();
    this.el.on('click',function() {
      alert('STARTED');
    });
  }
});

Details

There are a few implied behaviors to be covered.

  • You can get away without coding the renderText() method if your custom tag renders just like an HTML tag. In this case, you indicate the tag name to use as the attribute ‘htmlTag’ and rely on the default renderText() method.
  • You can invoke an overridden method using the syntax “this._super()”. This is part of the Class.js implementation.
  • The activate() method of the View Class makes sure this.el is set. It is normally set by the render() method, but if not, and the object has an ‘id’ attribute, then it is assumed that the ‘id’ matches the HTML tag and is used to select the HTML tag.

Conclusion

The Tags Framework is a light-weight, unobtrusive Framework that implements the missing paradigm that connects HTML and Javascript and lets them work together to more easily solve many of the programming tasks required for implementing a Client-Side MVC application.
